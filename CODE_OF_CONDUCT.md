# 行为准则

## 基本要求

本项目期望所有参与者保持尊重、建设性和围绕主题的沟通。

参与者应当：

- 用清晰、事实性的方式讨论技术决策。
- 在没有证据前，把问题优先看作普通失误。
- 把分歧聚焦在代码、文档、行为和项目范围上。
- 尊重项目文档中记录的双语内容模型和静态站约束。

参与者不应：

- 骚扰、威胁、侮辱或人身攻击他人。
- 未经明确许可公开他人的私人信息。
- 用无关争论干扰 issue 或 pull request。
- 在没有项目决策的情况下，强迫维护者添加评论、分析、CMS、搜索、草稿或项目详情页等受控功能。

## 执行

维护者可以编辑、隐藏、锁定或移除违反本准则的评论、issue、pull request 或其他贡献。重复或严重违反者可能被临时或永久限制参与。

## 报告

请私下向项目 owner 报告行为准则问题。不要使用公开 issue 报告敏感行为问题。
