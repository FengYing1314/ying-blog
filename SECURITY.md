# 安全政策

## 支持版本

当前仓库支持 `main` 和 `fy-blog` 分支。漏洞修复应按影响范围回补到对应分支。

## 报告漏洞

如果仓库托管平台支持 private security advisory，请使用私有安全通道报告。若尚未启用私有安全通道，请先联系项目 owner，不要在公开 issue 中发布漏洞细节。

报告时请包含：

- 受影响的路由、脚本或内容功能。
- 复现步骤。
- 预期行为和实际行为。
- 对静态输出、客户端交互或生成 metadata 的已知影响。

## 安全敏感区域

以下改动需要重点审查：

- Markdown 渲染。
- 生成 HTML。
- 外部链接处理。
- 构建脚本。
- RSS 和 sitemap 生成。
- 部署配置。
